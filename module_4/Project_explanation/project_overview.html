Alert System Logic

Multi-tiered alert mechanisms with configurable thresholds

üë§
Per-Person Alert

Triggered when an individual's cumulative time in the red danger zone exceeds the configured threshold. Enables tracking of specific personnel in hazardous areas.

IF person_red_zone_time > person_threshold
THEN log_alert(person_id, "Person Alert")
üö®
Zone Population Alert

Activated when the number of people currently present in the red zone exceeds capacity limits. Critical for preventing overcrowding in restricted areas.

IF current_red_zone_count > zone_threshold
THEN activate_zone_alert()
‚ö†Ô∏è
Overall Population Alert

System-level alert triggered when total detected individuals across all zones surpass the overall capacity threshold. Useful for venue-wide crowd management.

IF total_detected_people > overall_threshold
THEN system_alert("Overcrowding")

Local Setup Guide

Step-by-step installation and configuration instructions

1. Clone Repository

git clone https://github.com/your-username/crowd-monitoring-ai.git
cd crowd-monitoring-ai

2. Create Virtual Environment

# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate

3. Install Dependencies

pip install --upgrade pip
pip install -r requirements.txt

# Key packages:
# - ultralytics (YOLOv8)
# - flask, flask-jwt-extended
# - sqlalchemy, psycopg2
# - opencv-python
# - reportlab

4. Configure Environment

# Create .env file
DATABASE_URL=postgresql://user:pass@localhost/crowd_db
SECRET_KEY=your-secret-key-here
JWT_SECRET_KEY=your-jwt-secret
FLASK_ENV=development
DEBUG=True

5. Initialize Database

python
>>> from app import db
>>> db.create_all()
>>> exit()

# Or use migration tools
flask db init
flask db migrate
flask db upgrade

6. Run Application

python app.py

# Server starts at:
# http://127.0.0.1:5000

# For production:
gunicorn -w 4 -b 0.0.0.0:5000 app:app
